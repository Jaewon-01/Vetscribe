# VetScribe

보호자 커뮤니케이션 자동화 서비스
서비스 스펙 문서 v2.0
2025년 5월

---

## 1. 프로젝트 개요

### 1.1 한 줄 요약

한국 중소형 동물병원 수의사를 위한 보호자 커뮤니케이션 자동화 서비스. 예방접종 리마인드, 수술 전후 안내, 재내원 안내를 카카오톡으로 자동 발송하고, 안내문 작성을 AI가 보조한다.

### 1.2 피봇 히스토리

**아이디어 발전 과정**

- **v1.0** 보호자용 응급 진단 앱 → 보수적 설계 시 "병원 가세요"로만 귀결, 앱 사용 동기 약화
- **v1.5** 수의사용 AI 차팅 앱 (녹음 기반 SOAP note) → 실사용자 인터뷰에서 도입 허들 발견
- **v2.0** 보호자 커뮤니케이션 자동화 (현재) → 인터뷰에서 확인된 실제 페인포인트 기반

**실사용자 인터뷰 핵심 인사이트 (현직 수의사)**

- ✓ AI 차팅: "편하긴 한데, 녹음 동의 → 보호자가 녹취본 요구할 수 있어 법적 리스크"
- ✓ 문자 알림: "예방접종 리마인드를 직원이 수동으로 버튼 눌러 보내고 있음"
- ✓ 수술 안내: "간호사가 개인 휴대폰으로 보호자에게 직접 보내고 있음"
- ✓ CRM: "신규 제품 출시 시 기존 내원 고객 대상 메시지 자동화 원함"

### 1.3 타겟 유저

| 구분 | 설명 |
|---|---|
| **Primary** | 한국 중소형 동물병원 수의사/원장 (1~3인 운영, 하루 20~30마리 진료) |
| **Secondary** | 동물병원 간호사/직원 (안내문 발송 업무 담당) |
| **수혜자** | 보호자 (카카오톡 안내문 수신, 직접 앱 사용 없음) |

### 1.4 핵심 가치 제안

- 예방접종·수술·재내원 안내문을 AI가 자동 생성하고 카카오톡으로 발송
- 수의사가 직접 문자를 작성하거나 간호사가 개인 폰으로 보내는 수작업 제거
- 환자 정보(품종, 나이, 수술 종류) 기반 개인화된 안내문 — 단순 템플릿 아님
- 녹음 불필요 → 법적 리스크·도입 허들 없음

## 2. 시장 분석 및 차별화

### 2.1 한국 동물병원 EMR 현황

실사용자 인터뷰 및 리서치를 통해 확인한 주요 플레이어:

| 업체 | 제품 | 비고 |
|---|---|---|
| 우리엔 | 이프렌즈 / PMS365 | 국내 1위, 400곳+ 사용, 인터뷰 병원 사용 중 |
| 벳칭 | 플러스벳 | EMR + CRM 통합, 클라우드 기반 |
| 아이엠디티 | 클레어(CLAiR) EMR | 보호자 케어 가이드 자동발송 특허 보유 — 직접 경쟁 주의 |
| 인투벳 | 인투벳 EMR | 국세청 매출 자동 전송 인식으로 일부 병원 기피 |

### 2.2 VetScribe 포지셔닝

> **핵심 포지셔닝**
> "기존 EMR을 대체하지 않는다. 기존 EMR 위에 얹히는 보호자 커뮤니케이션 레이어다."
>
> EMR: 차트 기록·관리·청구 → VetScribe: 보호자에게 나가는 모든 메시지 자동화
> 경쟁이 아닌 보완재 → 우리엔 등 EMR 업체와 파트너십 대화 가능

## 3. 기능 스펙

### 3.1 핵심 플로우

```
[수의사/직원] 안내문 유형 선택
       ↓
환자 기본 정보 입력 (이름, 품종, 나이, 해당 항목별 추가 정보)
       ↓
Claude API → 개인화된 안내문 자동 생성
       ↓
수의사 확인·편집
       ↓
[카카오톡 발송] 또는 [복사 후 직접 발송]
```

### 3.2 안내문 유형 (MVP 범위)

**① 예방접종 리마인드**

- 접종일 D-7, D-1 자동 알림 생성
- 백신 종류(종합백신, 광견병, 켄넬코프 등)에 따라 문구 자동 분기
- 다음 접종일·접종 전 주의사항 포함

**② 수술 전 안내**

- 수술 종류(중성화, 슬개골, 발치 등) 선택 → 맞춤 주의사항 생성
- 금식 시간, 당일 준비물, 보호자 연락 방법 포함
- 수술 전날 발송 타이밍 설정 가능

**③ 수술 후 / 퇴원 안내**

- 수술 종류 + 처방약 정보 입력 → 집에서 해야 할 일 자동 생성
- 약 복용법, 식이 제한, 활동 제한, 상처 관리 포함
- 재내원 필요 신호(red flag) 자동 포함

**④ 재내원 리마인드**

- 다음 내원일 입력 → "몽이 내원일은 OO입니다" 자동 문자 생성
- 현재 직원이 수동으로 버튼 눌러 보내는 방식을 자동화

**⑤ CRM 메시지 (MVP 이후)**

- 신규 사료·간식·서비스 출시 시 기존 내원 고객 대상 맞춤 메시지
- 품종·나이·기저질환 기반 타겟팅

### 3.3 Out of Scope (MVP)

- 진료 중 녹음 기반 SOAP note — 녹음 동의 법적 리스크로 제외
- 실제 카카오톡 API 직접 연동 — UI상 발송 시뮬레이션으로 대체
- EMR 직접 연동 — 복붙 방식으로 MVP 진행, 이후 파트너십 검토
- 보호자 로그인·앱 화면
- 환자 DB / 차트 이력 관리

## 5. AI 프롬프트 스펙

### 5.1 수술 후 안내문 — System Prompt

```
당신은 한국 동물병원 수의사를 보조하는 AI입니다.
보호자에게 카카오톡으로 전송할 퇴원 안내문을 작성합니다.

규칙:
- 보호자가 이해하기 쉬운 쉬운 말 사용 (의학 용어 최소화)
- 친절하고 따뜻한 어조, 이모지 적절히 사용
- 약 복용법·식이·활동 제한·상처 관리·red flag 반드시 포함
- 500자 이내로 간결하게
- 카카오톡 줄바꿈 형식에 맞게
```

### 5.2 수술 후 안내문 — User Prompt

```
환자 정보:
- 이름: {patientName}
- 품종/나이: {breed} {age}세
- 수술 종류: {surgeryType}
- 처방약: {medications}
- 다음 내원일: {nextVisit}

위 정보를 바탕으로 보호자용 카카오톡 안내문을 작성해주세요.
반드시 포함: 약 복용법 / 주의사항 / red flag / 다음 내원일
```

### 5.3 예방접종 리마인드 — User Prompt

```
환자 정보:
- 이름: {patientName}
- 품종/나이: {breed} {age}세
- 접종 종류: {vaccineType}
- 접종 예정일: {vaccineDate}
- 발송 시점: D-{days}

접종 리마인드 카카오톡 메시지를 작성해주세요.
포함: 접종 안내 / 당일 주의사항 / 병원 연락처 안내
```

## 7. 비즈니스 스토리

### 7.1 시장 규모

- 한국 동물병원 약 5,000개 (농림축산식품부 기준)
- 중소형 (1~3인 운영) 약 80% → 타겟 SAM 약 4,000개
- 월 구독 3만원 기준 → 전체 전환 시 연 144억원 규모

### 7.2 수익 모델

| 플랜 | 가격 | 포함 기능 |
|---|---|---|
| 베이직 | 무료 (월 30건 제한) | 퇴원 안내, 재내원 리마인드 |
| 스탠다드 | 월 29,000원 | 전체 안내문 유형 + 무제한 |
| 프로 | 월 59,000원 | 스탠다드 + CRM 메시지 + 분석 |

### 7.3 도입 전략

- 프리티어로 도입 허들 낮추기 → 실사용 후 전환
- 우리엔·플러스벳 사용 병원에 "기존 EMR 그대로 쓰면서 추가"로 포지셔닝
- 수의사 커뮤니티(수의사 네이버 카페, 인스타그램) 바이럴 마케팅
- 초기 10개 병원 무료 파일럿 → 케이스 스터디 확보
